# WBS 일정 관리

시작일과 M/D(공수)를 입력하면 대한민국 공휴일·주말을 제외한 영업일 기준으로 종료일이 자동 계산되는 단일 HTML 파일 기반 WBS(작업분류체계) 관리 도구입니다. 서버나 설치 과정 없이 `wbs.html`을 더블클릭해서 바로 사용합니다.

## 로컬 사용법

1. `wbs.html`을 더블클릭해 브라우저로 엽니다.(https://butifriend.github.io/wbs/wbs.html)
2. 그룹/하위업무를 추가하고 시작일·M/D를 입력하면 종료일이 자동으로 채워집니다.
3. 입력한 데이터는 그 브라우저의 로컬 저장소(localStorage)에 자동 저장되어, 같은 브라우저로 다시 열면 이어서 작업할 수 있습니다.
4. 공휴일 목록, 엑셀(.xlsx)/CSV 다운로드, 인쇄 등은 상단 툴바에서 사용할 수 있습니다.

## 팀원과 공유하는 방법 (GitHub 기반 수시 동기화)

> GitHub은 버전관리 시스템이라 구글 시트처럼 여러 명이 **동시에** 실시간으로 입력하는 방식은 지원하지 않습니다. 대신 이 저장소를 통해 **작업 → 내보내기 → 커밋/푸시 → 팀원이 pull → 가져오기** 흐름으로 데이터를 주고받습니다.

데이터는 [`JSON 백업/wbs-data.json`](./JSON%20백업/wbs-data.json) 파일 하나로 관리합니다.

### 처음 시작할 때
```
git clone https://github.com/butifriend/wbs.git
```
그 다음 `wbs.html`을 열고, 상단 툴바의 **JSON 가져오기**로 `JSON 백업/wbs-data.json`을 불러오면 팀의 최신 데이터로 시작할 수 있습니다.

### 작업 후 팀에 공유할 때
1. `wbs.html`에서 작업 전, **JSON 가져오기**로 `JSON 백업/wbs-data.json`(최신본)을 먼저 불러와 시작하세요.
2. WBS 내용을 수정합니다.
3. 상단 툴바의 **JSON 내보내기**를 누르면 `wbs_YYYY-MM-DD.json` 파일이 다운로드됩니다.
4. 이 파일을 저장소의 `JSON 백업/wbs-data.json` 파일에 덮어씁니다(파일명을 `wbs-data.json`으로 맞춰주세요).
5. 커밋 후 푸시합니다.
   ```
   git add "JSON 백업/wbs-data.json"
   git commit -m "WBS 업데이트: <변경 내용 요약>"
   git push
   ```
6. 다른 팀원은 `git pull` 후 **JSON 가져오기**로 최신 내용을 불러옵니다.

### 충돌을 피하려면
- 동시에 여러 명이 편집하면 `wbs-data.json`에서 git 충돌이 날 수 있습니다. 편집 전 팀 채널 등에서 "지금 수정 중" 정도만 짧게 공유해도 충돌을 크게 줄일 수 있습니다.
- 충돌이 나면 `git pull` 후 최신 파일 기준으로 본인 변경사항을 다시 반영해 재작업하는 것이 JSON을 직접 손으로 병합하는 것보다 빠릅니다.

### 참고
- 공휴일 커스터마이징(직접 추가/삭제한 공휴일)은 브라우저별 로컬 저장소에만 남아 `wbs-data.json`에 포함되지 않습니다. 팀 전체가 같은 공휴일 목록을 쓰려면 각자 공휴일 관리에서 동일하게 추가해주세요.
- 엑셀(.xlsx)/CSV 다운로드는 그 시점의 스냅샷 내보내기용이며, 공유 동기화에는 `JSON 백업/wbs-data.json`을 사용합니다.
